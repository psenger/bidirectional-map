## Architecture

This is a zero-dependency npm package (`@psenger/bidirectional-map`) that exports a single class: `BiDirectionalMap`, which extends the native `Map`.

**Source:** `src/com/cngr/bidirectionalmap/index.js`  
**Dist:** `dist/index.js` (CommonJS, generated by Rollup)  
**Tests:** co-located at `src/com/cngr/bidirectionalmap/index.spec.js`

### How `BiDirectionalMap` works

It maintains a one-to-one symmetric relationship: calling `set(A, B)` stores both `A→B` and `B→A` as entries in the underlying `Map`. The overridden `set` method first removes any pre-existing relationships for either key or value before registering the new pair, preventing stale inverse mappings. Only `String` and `Number` types are supported as keys/values.

### README generation

`README.md` is generated — do not edit it directly. The source template is `.README.md`, which uses `@psenger/markdown-fences` section markers. `npm run docs` (via `build-readme.js`) stitches together the template, JSDoc extracted from `index.js`, and the `INSTALLATION.md` / `TUTORIAL.md` partials.

## Release workflow

Releases go via `main` branch. The flow is: PR from `dev` → `main`, then on `main` run `npm version patch|minor -m "..."`, then `npm publish --access public`. After publishing, merge `main` back into `dev`.

The `version` lifecycle script automatically builds and stages `dist/`, and `postversion` pushes the tag.

## Code style

StandardJS (`standard`) is the linter. ESLint is configured to skip `*.spec.js` files. Node ≥ 22 is required (`.nvmrc` pins this).
